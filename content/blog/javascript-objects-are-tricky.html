The trick is in how objects in Javascript are created. When you create a new plain object as const obj = {} in reality this equals to Object.create(Object.prototype). Prototypes are out of the scope of this post but they‚Äôre basically a Javascript way of implementing inheritance. Therefore, when you create an empty object, it already has some functions bound to it.

And thus, obj['constructor'] is already defined even in an ‚Äúempty‚Äù object, therefore the database complained because we didn‚Äôt set an integer count but actually a function () {} + 1.

To fix this I could create an object like this, but that‚Äôs a bit ugly and weird.
